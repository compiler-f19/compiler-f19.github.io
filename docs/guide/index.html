介紹

我們在這學期做了一個新嘗試，將作業的撰寫、繳交與 Git 結合，並將課程網站放上 GitHub 上使用，目的是提高各位的修課體驗，並嘗試與現今資訊世界的做事流程接軌。

如果有怎麼問題的話，也請各位不要害怕，到討論區發問，你是修課生，你會有的問題其他學生也會有，幫助你自己的同時也是在幫助其他同學，也可以讓授課老師知道課程有什麼事情需要調整。 我們做的改動相較過去帶來不少好處，在以下列出幾點讓大家更能理解這些背後的原因。

GitHub 網站的穩定性比 E3 高

身為交大的學生可能體驗過在作業死線來臨前 1 分鐘繳交作業，e3 直接直接死掉不回應的情況，或是在期末考週學校 E3 伺服器壞掉讓各位無法瀏覽。我們把課程的網站跟作業繳交直接託管在 GitHub 這種國際型企業上能大幅提升網站穩定性，讓大家能隨時瀏覽相關內容。我們也會利用 GitHub 提供的網站紀錄判斷是否有因為伺服器下線讓作業無法繳交的情況，直接延長死線的時間。

GitHub 當作你的遠端備份

你只需要一個指令就能將程式碼放上 GitHub，讓你能很簡單的備份自己的作業，降低誤刪自己作業檔案的風險。

強迫各位使用版本控管

或許你各位上大學唸資工從來沒聽過版本控管，或是單純沒機會使用。沒關係，這堂課程會讓你最低限度透過 Git 來練習版本控管，你可以拿來這個東西來輔助自己的未來的專案開發流程。 可能一開始用 Git 會有點不方便，甚至多餘，但你都已經繳學費來學習了，不妨多花點心力來學習一些可以輔助你工作的工具吧！

組織導覽

我們把這個課程中所有內容都放在一個Github組織下: compiler-f19

+ compiler-f19                       ——  課程開的 GitHub Organization
    + Discussion                       ——  討論版
    + Compiler-f19                  ——  課程內容
    + Compiler-f19-private  ——  不公開的教學內容

這是我們所開放，修課生必須要知道的 Repo (Repository)。 而 Compiler-f19，Compiler-f19-private 兩個 Repo 的內容差別在於後者的內容屬於修課生才能擁有的內容，比如：每次作業的邀請代碼、考古題連結、整理好的某些筆記內容、考試的答案解析。同學必須要申請加入組織才有權限瀏覽。

加入流程

註冊 GitHub 帳號

請注意，這個帳號並不會是課程帳號，你只是在 GitHub 上註冊一個代表你個人的帳號，並且暫時加入我們的課程組織而已。這個帳號會一定程度的代表你個人，建議先仔細思考好帳號的名稱。 比如說 nctuGreatAgain, UCMaoChung5566, dutyForOurCountry 都是不推薦的名稱。

申請 GitHub 帳號連結： 點我

填表單

請在我們做的 Google 表單中填上你的資料 (必須要登入)，讓我們能夠將各位同學的 GitHub 帳號跟真實身份關聯在一起，在助教收到表單並確認為修課生後，會將各位填入的Github帳號加入組織，而GitHub 網站會依此發出邀請信。各位收到信並且按完接受後就算正式加入這堂課程了。👍

加退選期限過後會再用修課名單來更新組織裡的學生，不幸沒選上卻想旁聽留在組織內的同學請知會老師與助教一聲。

表單的 Google 連結：點我

加入組織

這個步驟是助教會做的，你只要上Github網站確認你有被加入組織即可。助教會在（）公告已經將哪些同學加入組織中，若你發現助教已經將你加入組織而你的Github帳號並未顯示你已被加入組織，請來信詢問助教。

開啟通知

開啟通知後在你參與的Repo下有任何改動都會在Github網站通知你。

Personal Settings > Notifications > Participating > Web[打勾]

申請作業0，熟悉環境

首先點擊https://github.com/compiler-f19/Compiler-f19-private/tree/master/Homework拿取作業的邀請碼，若你發現點進網站後顯示404 page not found，代表你還沒被加入組織中，請等加入組織後再次嘗試。
